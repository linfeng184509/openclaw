## 1.1 什么是 OpenClaw

**OpenClaw** 是一个**个人 AI 助手网关**，让你能够在自己设备上运行属于自己的 AI 助手。

与依赖云服务的 AI 助手不同，OpenClaw 采用**本地优先**（Local-first）的设计理念。它在你自己的设备上运行，通过你已有的消息渠道（WhatsApp、Telegram、Slack、Discord、Signal、iMessage 等）与你交互，同时可以控制你的本地设备（macOS、iOS、Android）执行实际操作。

### 核心特点

| 特点 | 说明 |
|------|------|
| **本地运行** | Gateway 运行在你的设备上，数据不出域 |
| **多渠道集成** | 支持 20+ 消息平台作为交互界面 |
| **设备控制** | 可直接操作 macOS/iOS/Android 设备功能 |
| **开源透明** | 代码完全开放，可自行审计和修改 |
| **高度可定制** | 支持插件、技能、自动化的深度定制 |

### 1.1.1 OpenClaw 的名字来源

OpenClaw 的名字来源于其吉祥物——一只名叫 **Molty** 的太空龙虾（Space Lobster）🦞。项目名称体现了其开源（Open）和"爪子"（Claw）的双重含义，象征着：

- **Open**：开放源码、开放协议、开放生态
- **Claw**：像龙虾爪子一样灵活地"抓取"各种工具和服务

## 1.2 OpenClaw 的核心价值

### 1.2.1 数据隐私与主权

在云服务主导的时代，个人数据往往存储在第三方服务器上。OpenClaw 反其道而行之：

- **数据本地存储**：所有会话历史、配置、凭证都存储在本地（`~/.openclaw/`）
- **无云端依赖**：核心功能不依赖任何云服务（除了你选择的 LLM 提供商）
- **自主控制**：你决定哪些数据可以发送、发送到哪里

```
┌─────────────────────────────────────────────────────────┐
│                    你的设备                              │
│  ┌─────────────┐    ┌─────────────┐    ┌─────────────┐ │
│  │   Gateway   │───▶│   本地数据   │    │   凭证存储   │ │
│  │  (控制平面)  │    │  (SQLite)   │    │  (Keychain) │ │
│  └─────────────┘    └─────────────┘    └─────────────┘ │
└─────────────────────────────────────────────────────────┘
         │
         ▼ (仅与你选择的 LLM 提供商通信)
┌─────────────────┐
│  LLM Provider   │
│  (OpenAI/Anthropic/本地) │
└─────────────────┘
```

### 1.2.2 统一的消息中枢

现代人在多个消息平台上活跃，OpenClaw 将它们统一到一个 AI 助手中：

| 渠道类型 | 支持的平台 |
|----------|------------|
| 即时通讯 | WhatsApp, Telegram, Signal |
| 团队协作 | Slack, Discord, Microsoft Teams, Mattermost |
| 苹果生态 | iMessage, BlueBubbles |
| 开源协议 | Matrix, Nostr, IRC |
| 其他 | Google Chat, LINE, Feishu, Zalo, Twitch 等 |

所有渠道的消息都可以路由到同一个 AI 助手，或者根据规则路由到不同的专用助手。

### 1.2.3 真正的设备控制

不同于只能"聊天"的 AI 助手，OpenClaw 可以执行实际操作：

- **浏览器控制**：打开网页、点击按钮、填写表单、截图
- **文件操作**：读取、写入、编辑本地文件
- **代码执行**：运行脚本、编译代码、执行命令
- **媒体处理**：拍照、录屏、播放音频
- **自动化任务**：定时任务、Webhook、邮件监控

```bash
# 示例：让 AI 助手发送消息
openclaw message send --to +1234567890 --message "Hello from OpenClaw"

# 示例：让 AI 助手执行任务
openclaw agent --message "分析项目目录结构并生成报告"
```

### 1.2.4 灵活的模型选择

OpenClaw 支持多种 LLM 提供商，不会被锁定在单一供应商：

| 提供商 | 支持模型 | 认证方式 |
|--------|----------|----------|
| OpenAI | GPT-4, GPT-4o, o1, o3 | API Key / OAuth |
| Anthropic | Claude 3/4/4.5 系列 | API Key / OAuth |
| Google | Gemini 2.0/2.5 系列 | API Key / OAuth |
| 本地模型 | Ollama, vLLM, LM Studio | 本地部署 |

支持**模型故障转移**（Model Failover），当主模型不可用时自动切换到备用模型。

## 1.3 应用场景与案例

### 1.3.1 个人助手场景

**场景 1：日常任务管理**
```
你（通过 WhatsApp）：提醒我下午 3 点开会
OpenClaw：已创建提醒，下午 2:55 会通知您。需要我提前准备会议材料吗？
```

**场景 2：信息检索**
```
你（通过 Telegram）：查一下最新的 React 版本
OpenClaw：[打开浏览器搜索] → React 19.0 已于 2025 年发布，主要特性包括...
```

**场景 3：代码辅助**
```
你（通过 Slack）：帮我写一个 Python 脚本，批量重命名文件
OpenClaw：[生成代码] → 已创建 rename_files.py，需要我执行测试吗？
```

### 1.3.2 家庭自动化场景

```
你（通过 iMessage）：家里温度怎么样？
OpenClaw：[查询 HomeKit] → 客厅 22°C，卧室 20°C。需要调节吗？
你：调高到 24 度
OpenClaw：[执行命令] → 已将恒温器设置为 24°C
```

### 1.3.3 开发工作流场景

```
你（通过 Discord）：检查最近提交的 CI 状态
OpenClaw：[调用 GitHub API] → 最近 3 个提交中，2 个通过，1 个失败。失败原因：测试超时。
你：帮我创建修复分支
OpenClaw：[执行 Git 命令] → 已创建分支 fix/ci-timeout，需要我分析失败日志吗？
```

### 1.3.4 团队协作场景

```
团队成员 A（通过 Slack）：@bot 生成项目进度报告
OpenClaw：[分析 Jira/GitHub] → 本周完成 12 个任务，3 个进行中，1 个阻塞。详细报告：[链接]
```

## 1.4 与其他 AI 工具的对比

### 1.4.1 与云服务 AI 助手对比

| 维度 | OpenClaw | 云服务 AI 助手（如 ChatGPT、Claude.ai） |
|------|----------|----------------------------------------|
| 数据位置 | 本地存储 | 云端存储 |
| 隐私控制 | 完全自主 | 依赖服务商政策 |
| 消息渠道 | 20+ 平台可选 | 仅限官方应用/网站 |
| 设备控制 | 支持本地操作 | 不支持 |
| 定制能力 | 完全开源可修改 | 封闭系统 |
| 成本 | 仅 LLM 费用 | 订阅费 + 可能的 API 费用 |
| 离线能力 | 部分支持（本地模型） | 完全依赖网络 |

### 1.4.2 与其他开源 AI 框架对比

| 框架 | 定位 | 消息渠道 | 设备控制 | 部署难度 |
|------|------|----------|----------|----------|
| **OpenClaw** | 个人 AI 助手网关 | 20+ | 支持 | 中等 |
| LangChain | AI 应用开发框架 | 需自行集成 | 需自行开发 | 较高 |
| AutoGen | 多代理研究框架 | 有限 | 有限 | 较高 |
| Open Interpreter | 本地代码执行 | 有限 | 部分支持 | 较低 |

### 1.4.3 OpenClaw 的独特优势

1. **消息渠道优先**：在你已经使用的平台上与 AI 交互，无需学习新界面
2. **设备原生能力**：深度集成 macOS/iOS/Android 系统功能
3. **单一用户设计**：专为个人使用优化，无需复杂的多用户管理
4. **渐进式学习**：从简单聊天到复杂自动化，按需使用
5. **社区生态**：活跃的开发者和用户社区，持续贡献插件和技能
